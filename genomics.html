Genome Assembly

Genome assembly is the process of constructing the original genome from raw, fragmented DNA sequences alone.

The Genome Assembly Pipeline.

N/B: This pipeline is created and tested with Unix (Ubuntu 20.0 LTS distro). It should work on any Unix OS as long as the right softwares are installed. See appendix for list. The dataset is an abridged form of the dataset uploaded at ENA. See source in appendix.

1. Create a new folder and check into the folder

mkdir nimr_workshop
cd nimr_workshop 

2. Download your dataset (If you have it on another folder, move it to the new folder we created in step 1)

$ wget ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR957/SRR957824/SRR957824_1.fastq.gz
$ wget ftp://ftp.sra.ebi.ac.uk/vol1/fastq/SRR957/SRR957824/SRR957824_2.fastq.gz

$ fastqc SRR957824_500K_R2.fastq.gz

6. Based on your result in step 4, determine whether you have to Perform demultiplexing to remove duplicate identifiers using picard. In our case, we do not have any issue with duplicate reads or RNA artefacts.

Variant Calling

Now that we have connected fragments of nucleotides into a genome assembly of over thousands of thousands of nucleotides, what can we do with the millions of nucleotides? Let us assume they are from the same population/species (which is true for our case), it makes sense to try to identify what makes the different samples unique or different on its own. But if we find similar variations between a subset in a population, that makes it even more interesting!

How can this be significant? Think of it this way, assuming you have a dataset containing the complete genome sequence of 50 diseased patients and 50 normal patients. Is it possible to use the genomic data alone to declassify the patient states and identify the gene-level variants that determine the disease state and the impact of such variants on the disease process? 

Variant Calling for Diploid Organism: The Pipeline.

N/B: This pipeline is created and tested with Unix (Ubuntu 20.0 LTS distro). It should work on any Unix OS as long as the right softwares are installed. See appendix for list.

1. Create a new folder 

$ mkdir variant_call

2. Using the same trimmed files from our previous work, let us build a genome map which we will use to identify variants downstream. To identify variants, there must be an assumed truth-base to compare with. Bowtie identifies that in the form of a graph. So we need to create an index file (graph) which we will then use to create a map. First, Let’s download the E.Coli K-12 genome from NCBI (https://www.ncbi.nlm.nih.gov/nuccore/U00096.2). 

#visit the NCBI link and export the genbank output as a fasta file
$ bowtie2-build e_coli_k_12.fasta e_coli_k_12

3. Use the index to build a genome map for each sample data and specify the output in the variant_call folder.

$ bowtie2 -x e_coli/e_coli_k_12 -1 SRR957824_500K_R1_TRIM.fastq.gz -2 SRR957824_500K_R2_TRIM.fastq.gz -S SRR957824_500K.sam --no-dovetail

4. We are working with a prokaryotic genome with less genetic complexity. We will use freebayes to identify 

$ samtools view -S -b SRR957824_500K.sam > SRR957824_500K.bam
$ samtools sort SRR957824_500K.bam -o SRR957824_500K.sorted.bam
$ samtools index SRR957824_500K.sorted.bam
# A preview into the variants
$ samtools tview SRR957824_500K.sorted.bam e_coli_k_12.fasta

5. Call variants with this file or its compressed form (.bam) using freebayes. Output from freebayes usually requires further annotation.

$ freebayes -f e_coli_k_12.fasta SRR957824_500K.sorted.bam variant_ecoli.vcf

6. Call variants with samtools mpileup to generate raw SNPs

$ samtools mpileup -uD -f e_coli_k_12.fasta SRR957824_500K.sorted.bam | bcftools view - > variant_ecoli_mpileup.vcf

# https://igv.org/app/

$ vcfstats
